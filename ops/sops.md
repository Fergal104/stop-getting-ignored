# Standard Operating Procedures

## Daily 20-Minute Operating Loop

1. Publish one post from `content/launch-posts.md`.
2. Check X and Threads replies for any obvious session or posting issues.
3. Check Gumroad visits and downloads.
4. Log the day in `tracking/kpi-tracker.csv`.
5. If a post earns engagement, repurpose it into a longer Threads variation.

## Weekly 60-Minute Batch

1. Review the previous 7 days of metrics.
2. Identify the best-performing hook.
3. Generate 10 new posts using `ops/prompts.md`.
4. Rewrite the best two posts into longer Threads-style explanations.
5. Add one new personalization example to the paid kit if a common objection appears.

## Posting Rules

- Lead with the idea, not the product.
- Use the free pack CTA lightly.
- Do not argue in comments.
- Do not promise conversions or revenue.
- If a platform dislikes direct links, move the CTA to profile bio.

## Funnel Rules

- Do not add more platforms before the landing page and Gumroad product are live.
- Do not raise the price before 5 sales or strong conversion evidence.
- Do not build a newsletter until 25 to 50 emails exist.

## Pivot Rules

- If there are fewer than 50 visitors by day 7, improve distribution volume and hooks.
- If there are visits but no lead magnet downloads, fix the landing-page CTA and hero.
- If there are downloads but no checkout clicks, improve the bridge from free to paid.
- If there are checkout clicks but no sales, improve the Gumroad page or price framing.
- If there is no sale by day 14, change the niche angle before adding complexity.
